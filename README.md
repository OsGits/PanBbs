# PanBbs - 网盘资源聚合

一个轻量级的网盘资源聚合展示系统，从远程 API 拉取数据并分类展示，支持搜索、滚动加载、密码一键复制等功能。

## 项目来源

本项目是基于开源网盘搜索工具 [PanSo](https://github.com/fish2018/pansou) 的**扩展前端**。

- **PanSo** 提供了网盘资源搜索的后端 API 服务，负责数据抓取与检索
- **PanBbs** 在 PanSo 的基础上，构建了一套完整的前端展示层，提供更友好的浏览、搜索、分类体验

两者关系：

```
PanSo（后端 API）  ←   PanBbs（扩展前端）
   │                        │
   │ 提供 /api/search        │ 调用 API + 本地缓存
   │ 负责搜索与爬取          │ 分类展示 + 滚动加载 + 搜索 UI
   │                        │
   └──────── 配合使用 ────────┘
```

## 功能

- **资源聚合展示** — 从远程 API 抓取数据，按类型（115 / 光压 / 夸克）分类展示
- **搜索模块** — 通过弹窗输入关键词，直接调用远程 API 搜索，不落盘本地
- **滚动加载** — 首屏展示 10 条，滚动到底自动追加，滑到底自动刷新获取新数据
- **密码一键复制** — 点击密码区域即可复制到剪贴板（支持 Clipboard API 及 fallback）
- **毛玻璃搜索弹窗** — 半透明毛玻璃效果，支持回车搜索、ESC / X 按钮关闭
- **骨架屏加载** — 首屏加载时显示骨架屏动画，提升感知体验
- **回到顶部** — 右下角浮动按钮，滚动超过 500px 时显示
- **全局加载遮罩** — 数据刷新时显示遮罩，防止重复操作
- **静态资源版本控制** — 通过 `version.php` 统一管理 CSS/JS 缓存版本号

## 目录结构

```
PanBbs/
├── index.php              # 路由入口（home / api / search）
├── api.php                # 工具函数库（数据获取、解析、展平、存储）
├── ting.php               # 数据抓取触发脚本（拉取远程数据 → 存入 oss.json）
├── version.php            # 静态资源版本号（刷新浏览器缓存）
├── data/
│   └── index.php          # 目录占位（oss.json 由 ting.php 自动生成）
└── template/
    ├── home.php           # 前端模板页面（HTML 结构 + 数据注入）
    ├── app.js             # 前端交互脚本（滚动加载、搜索、密码复制等）
    └── style.css          # 样式表（现代简约风格 + 毛玻璃效果）
```

## 架构与数据流

### 路由分发 (`index.php`)

| Action | 说明 |
|--------|------|
| `?a=home`（默认） | 渲染前端模板页面 |
| `?a=api&type=xxx` | 返回本地 `data/oss.json` 中指定类型的数据（JSON） |
| `?a=search&kw=xxx` | 调用远程 API 搜索，解析展平后直接返回 JSON，**不存储到本地** |

### 数据抓取 (`ting.php`)

- 访问 `ting.php` 触发从远程 API 拉取全量数据
- 按类型分类后，按 URL 去重，合并写入 `data/oss.json`
- 每种类型最多保留 100 条记录（循环覆盖）
- 支持 `?debug=1` 调试模式，逐步排查抓取问题

### 前端数据加载

1. **首次访问** — `home.php` 将本地 `oss.json` 数据注入 JS 变量 `PANBBS_DATA`
2. **滚动到底** — 当前数据全部渲染完毕后，触发 `ting.php` 拉新数据，再通过 `?a=api` 重新加载
3. **搜索** — 弹窗输入关键词，直接请求 `?a=search&kw=xxx` 获取远程结果并展示

### 远程 API 格式

项目对接的远程 API 地址：`http://127.0.0.1:8010/api/search`

参数：
- `kw` — 搜索关键词（空字符串拉取全量）
- `conc=30` — 并发数
- `res=all` — 返回全部结果

返回的 JSON 结构支持多种格式（`data.results` / `data.list` / `list` 等），由 `parseRemoteData()` 自动兼容。

## 注意事项

1. **远程 API 依赖** — 项目依赖 `http://127.0.0.1:8010/api/search` 接口，部署前需确保该服务可用。如需更换地址，修改 `ting.php` 和 `index.php` 中的 `$remoteApi` 变量。

2. **数据目录权限** — `data/` 目录需要 PHP 写入权限，`ting.php` 运行时会在此目录生成 `oss.json`。

3. **目标类型** — 当前仅处理 `115`、`guangya`、`quark` 三种网盘类型，其他类型的链接会被自动过滤。修改 `$targetTypes` 数组可调整。

4. **搜索不落盘** — 搜索请求走 `?a=search` 直接从远程 API 获取并返回，不会写入本地 `oss.json`，确保本地数据干净。

5. **超时处理** — 前端请求超时设为 60 秒，超时仅提示不中断请求；数据抓取 `ting.php` 的 cURL 超时为 30 秒。

6. **UTF-8 修复** — `api.php` 中的 `fixUtf8()` 函数会自动修复远程返回数据中的非法 UTF-8 字符，防止 JSON 解析失败。

7. **缓存刷新** — 修改 CSS/JS 后，更新 `version.php` 中的版本号即可强制浏览器刷新缓存。

8. **最大记录数** — 每种类型最多保留 100 条记录（在 `ting.php` 中配置 `$maxRecords`），超出部分会被自动截断。

## 部署

将项目放置到 PHP 支持的 Web 服务器目录下，确保：

- PHP 版本 ≥ 7.0
- 允许 `file_get_contents()` 远程访问或已安装 cURL 扩展
- `data/` 目录可写

首次访问前，先访问一次 `ting.php` 拉取初始数据，然后访问首页即可正常使用。
