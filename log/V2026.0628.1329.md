# 更新日志

## V2026.0628.1329

1. 前端暗色/亮色主题切换
   - 新增右下角主题切换浮动按钮，位于版权按钮上方，点击可在日间/夜间模式间切换
   - 日间模式显示太阳图标，夜间模式显示月亮图标，切换带旋转动效
   - 暗色主题提供完整配色方案（深色背景、柔和文字、调整后的强调色），覆盖首页、搜索弹窗、详情弹窗、卡片、标签等所有区域
   - 用户主题偏好自动存入 localStorage（key: `panbbs_theme`），刷新后保持选择

2. 后台新增前端默认色彩设置
   - SEO 设置面板底部新增「前端默认色彩」下拉框，可选日间模式或夜间模式
   - 保存后写入 `data/data.php` 的 `$defaultTheme` 变量，作为新用户首次访问时的默认主题
   - 用户手动切换后优先使用本地存储偏好，后台设置仅影响无本地偏好的访问

3. 修改前端详情页弹窗已知BUG
   - 详情页弹窗在移动端浏览器中，点击「关闭」按钮时，页面会向下滚动
   - 已修复，详情页弹窗在移动端浏览器中，点击「关闭」按钮时，页面不会向下滚动
   - 弹窗框架的尺寸计算逻辑有问题，优化

4. 还定义标准属性“line-clamp”以实现兼容性
   - CSS 中使用了 -webkit-line-clamp 但没有同时定义标准属性 line-clamp，已添加标准属性 line-clamp: 2，与 -webkit-line-clamp: 2 并列，确保在支持标准属性的浏览器中也能正常工作。

5. 后台：admin.php?a=version 页面中，更新日记对md格式转换不完美，需要增加对代码的转换
   - 第 74 行的代码块正则 `/```(\w*)\n?(.*?)```/s` 中的 `\n?` 只匹配一个可选换行，但如果代码块内包含换行后的 HTML 实体 `&lt;br&gt;`（因为先做了 `htmlspecialchars`），会导致匹配不完整。另外，`simpleMarkdownToHtml` 函数对代码块的处理更好，应该直接复用它。
   - **之前**：版本页面内嵌了一套简陋的正则 Markdown 转换，代码块正则 `/```(\w*)\n?(.*?)```/s` 的 `\n?` 只能匹配一个换行，导致多行代码块转换不完整
   - **现在**：直接复用 `admin.php` 中已定义的 `simpleMarkdownToHtml()` 函数（该函数用逐行解析方式处理代码块，支持多行），同时使用 `.readme-body` 样式类获得统一的代码块深色主题、行内代码高亮等视觉效果
