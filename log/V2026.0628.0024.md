# V2026.0628.0024 更新日记

## 1. 修改账号密码功能增强

### 问题
系统设置中的「修改密码」面板仅支持修改密码，无法修改账号名。

### 改动

**前端 - `admin/page_settings.php`**
- 面板标题从「修改密码」改为「修改账号 / 密码」
- 新增「新账号」输入框，支持修改登录账号名
- 新账号和新密码均可选填，至少填写一项
- 原密码字段标注为「验证身份」
- 新增前端校验：账号格式（2-20位字母数字下划线）、密码长度≥6位
- 底部添加提示：忘记账号密码可在 `data/data.php` 中手动修改 `$accounts` 数组

**后端 - `admin/actions.php`**
- `adminHandleChangePassword()` 函数重写，支持三种场景：
  - 仅修改密码（新账号留空）
  - 仅修改账号（新密码留空，沿用原密码）
  - 同时修改账号和密码
- 账号名变更时自动同步更新 `$_SESSION['admin_username']`
- 修改 `data/data.php` 时正确替换旧账号条目为新账号条目

## 2. 版本页面显示 GitHub Releases 更新日志

### 问题
版本管理页面只显示本地 log 目录的更新日记文件列表，无法直接看到 GitHub Releases 的更新内容。

### 改动

**`version.php`**
- 新增获取 GitHub Release 的 `body` 字段（Markdown 格式更新内容）
- 定义 `PANBBS_RELEASE_BODY` 常量

**`admin.php`**
- 读取 `PANBBS_RELEASE_BODY` 并传递给 `adminShowVersion()`

**`admin/page_version.php`**
- 新增「最新发布更新日志」面板，直接展示 GitHub Releases 的 Markdown 内容
- 支持简单 Markdown → HTML 转换（标题、列表、代码块、加粗等）
- 底部附带「查看全部 Releases →」链接
- 移除本地更新日记文件列表面板
